## 1. 背景

因科特 G23 v2 是一款使用 PIXART 传感器的 2.4G 无线游戏鼠标，官方提供了一个基于 WebHID 的网页驱动（https://incott.net/mouse/），用于配置鼠标参数。

本文档记录了对该网页驱动源码的逆向分析过程及结果。

---

## 2. 设备基本信息

| 项目 | 值 |
|------|------|
| 鼠标型号 | 因科特 G23 v2 |
| 传感器 | PIXART |
| Vendor ID (VID) | `0x093A` |
| Product ID (PID) | `0x522C`（无线）/ `0x622C`（充电中） |
| USB 接口 | IF=2, Usage Page=`0xFF05` |
| HID Collection | **Col02**（必须使用第二个 Collection） |

---

## 3. HID 接口选择

该鼠标在 Windows 下会注册多个 HID 接口：

| 接口 | Usage Page | 用途 |
|------|-----------|------|
| IF=1 | `0xFF00` | 标准鼠标输入（不可用于配置） |
| IF=2 Col01 | `0xFF05` | Vendor 接口（不稳定） |
| IF=2 **Col02** | `0xFF05` | **正确的配置接口** ✓ |

> **重要提示**：必须使用 `UP=0xFF05` 的第二个 Collection（Col02）进行通信，其他接口会导致 `read error` 或返回错误数据。

## 4. 通信协议

### 4.1 读取状态命令

```
Report ID: 0x09
发送: [0x09, 0x89, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]  (9字节)
读取: get_feature_report(0x09, 64)
```

### 4.2 响应格式

原始响应示例（64 字节）：
```
09 89 64 01 00 02 00 01 5A 00 00 ...
```

| 偏移 | 字节 | 含义 | 示例值 |
|------|------|------|--------|
| buf[0] | 0x09 | Report ID | 0x09 |
| buf[1] | 0x89 | 命令回显（验证用） | 0x89 |
| buf[2] | - | **电量百分比**（>100 时减128=充电中） | 0x64 = 100% |
| buf[3] | - | 配置组/档位号 | 0x01 |
| buf[4] | - | **回报率索引** | 0x00 = 1000Hz |
| buf[5] | - | **DPI 索引** | 0x02 = 1600 DPI |
| buf[8] | 0x5A | 未知（可能为电压/温度等） | 0x5A = 90 |

### 4.3 DPI 预设表

| 索引 | DPI |
|------|-----|
| 0 | 400 |
| 1 | 800 |
| 2 | 1600 |
| 3 | 2400 |
| 4 | 3200 |
| 5 | 6400 |

## 6. 验证结果

使用 Python `hidapi` 库测试：

```python
import hid, time

VID = 0x093A
PIDS = [0x522C, 0x622C]

# 找到 Col02 接口
devices = hid.enumerate(VID, 0)
paths = [d['path'] for d in devices 
         if d['vendor_id']==VID and d['product_id'] in PIDS 
         and d.get('usage_page',0)==0xFF05]
path = paths[1] if len(paths) >= 2 else paths[0]

# 读取状态
h = hid.device()
h.open_path(path)
h.send_feature_report([0x09, 0x89, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00])
time.sleep(0.05)
resp = h.get_feature_report(0x09, 64)
h.close()

# 解析
battery = resp[2]
dpi = [400,800,1600,2400,3200,6400][resp[5]]
hz = [1000,500,250,125,8000,4000,2000][resp[4]]
print(f"电量:{battery}%  DPI:{dpi}  回报率:{hz}Hz")
```

输出：
```
电量:100%  DPI:1600  回报率:1000Hz
```

## 7. 注意事项

1. **不要向未知接口发送随机数据**，可能导致鼠标配置被写入固件（需通过官方驱动恢复出厂设置才能修复）
2. **2.4G 无线鼠标**无法通过 Windows 标准 API（WMI/PowerShell）读取电量，必须使用厂商私有 HID 协议
3. 每次读取后建议**关闭设备句柄**，避免阻塞其他程序访问
4. 电量变化很慢，**1 小时检测一次**足够
